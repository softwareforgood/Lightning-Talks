Password Security

Passwords: Used to validate and secure resources

Easiest Password Check

Store a Bob's password in a file. When verifying authentication, check the value he gives us to what is stored in that file.

Bob wants to prove to Alice that he is indeed Bob

Alice asks for some form of proof

Bob hands over his password

Alice confirms that the password she is given matches the password she has on file for Bob

Brute Forcing

Just Keep Guessing

Combating BF Attacks

Add things like:

  • Guess limits
  • Limited attempts
  • Lockout IP address after too many guesses
  • Reset password, email to user

HTTP Basic Auth

Base64 encoded username and password are sent as a header

Base64 encoding is NOT secure

            
Authorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==
Base64.decode('QWxhZGRpbjpvcGVuIHNlc2FtZQ==') >> Aladdin:open sesame

Session Cookies

Piece of data the user's browser sends back with each request after logging in.

document.cookie="certified=Alice's_notarized_letter"

Can still be intercepted/spoofed on HTTP only traffic

Wireshark

Watch all traffic on local network

Non-HTTPS traffic can be monitored/intercepted

Copy session cookies and load them into your browser! You're now that user!

FireSheep

Wireshark on Crack

Double click a user to authenticate as them for the selected service

Why HTTPS is Important

Secures network traffic, preventing Wireshark et al from seeing your information

Required by law (in most states) when processing customer credit card information

See: PCI compliance

Hashing

One-way math problems

Easy to calculate, hard to reverse

Hashed Passwords

Used to secure passwords everywhere

Rails web apps, mobile apps, desktop OSs, etc

Prevent passwords from being lost of the password 'list' is stolen from Alice

Windows Password Tables

Hashes user passwords and places them in a table

Limited to 2x7-character chunks

Cracking Time

Each character added makes it exponentially harder to crack

Commercial products can guess 2.8B passwords/sec

Password Entropy

Easy to Use

Hard to Crack

correcthorsebatterystable takes far longer to guess than Y0f$za51

Rainbow Tables

Pre-calculated hashes

Hashes lists of passwords, stores the original password and has in a database to be looked up later

Can be gigabytes to terabytes in size

Storage is cheaper than time

Google MD5


$ md5 -s "password"
MD5 ("password") = 5f4dcc3b5aa765d61d8327deb882cf99

Salting

Add some flavor to your passwords

md5(user_password + 'pepper') = d89eddeec748c49d5add2f8f347b8899

Salting in Rails

str = [password, salt].flatten.compact.join Digest::MD5.hexdigest(str)

Devise gem uses bcrypt by default, which is far stronger than md5

In Summary

  • Always use HTTPS
  • Salt & Hash Passwords
  • Prefer password length over "complexity"
